1.1. Describe the difference between authentication and authorization.

Authentication is confirming to the system or host that you are who you say you are; authorization is having the rights to do some task or access some data on the host based normally on who you are.

1.2. Define non-repudiation.

Non-repudiation is the ability to ensure that a party to a contract or a communication cannot deny the authenticity of their signature on a document or the sending of a message that they originated.

1.3. List three types of 'something you know' which can be used in authentication.

  1. Password

  2. PIN

  3. Username

  4. Passphrase

1.4. What topics might you expect to see covered in a general security policy?

  1. Staff roles and responsibilities including system administrators

  2. Password policy and protection

  3. Network connectivity and access (internal, external, Internet), and monitoring

  4. System access control privileges including special administrator privileges

  5. Information backup

  6. Contingency planning and disaster recovery

  7. Physical security

1.5. List two types of "something you have" which can be used in authentication.

  1. Token

  2. Smart-card

  3. One-time password list

1.6. What are two drawbacks of biometrics?

  1. Cost

  2. Inaccuracy (not 100% accurate)

  3. Privacy issues

  4. Hardware issues

1.7. When you browse to a secure web server, does that server present you with a digital signature or a digital certificate?

  1. Digital Certificate

1.8. List three of the normal fields which are present on a digital certificate.

  1. Owner's public key

  2. Owner's name

  3. Expiration date of the public key

  4. Name of the issuer (the CA that issued the digital ID)

  5. Serial number of the digital ID

  6. Digital signature of the issuer

1.1. Name the steps of the SKiP Method.

A Survivable Functional Unit is:

  • Vendor Provides

  • Harden/Secure

  • Prepare

  • Detect

  • Respond

  • Improve

  • Repeat

1.2. Explain the difference between best practice and best practical.

Best Practice is the best way to do something. Best Practical is a solution that is practical given the enterprise's constraints.

1.3. Explain what the vendors provide and why.

1.4. Define an Air-Gapped Test Environment.

Characteristics: few computer systems, physically disconnected from any other network, replicated SFUs in the AGTE, mission is to support building new SFUs.

1.5. For the Harden/Secure Step:

a. What is the purpose?

Harden and secure the computer system or network infrastructure components against known threats.

b. What are the products?

Systems that have known problems removed and are configured for the enterprise.

c. What are the nine practice areas?

  1. Install minimum essential OS features

  2. Install known and applicable patches

  3. Install most secure and current versions of system applications

  4. Replace applications that contain known vulnerabilities

  5. Install and replace applications that support other SFUs

  6. Install applications to support other steps in SKiP

  7. Removed all privileged and lenient object access

  8. Install and configure OS features and applications that support organizational policy and procedures

  9. Install and configure all OS features and applications that support risk management

d. What is the next step and what is the transition?

Prepare; Completion of the all of the nine practice areas

1.6. For the Prepare Step:

a. What is the purpose?

To recognize when new vulnerabilities have been or are being exploited.

b. What are the products?

A computer system or network infrastructure component whose routine actions have been characterized and this characterization information has been stored for later reference

c. What is Characterization?

Learning how a computer system or network infrastructure component operates normally.

d. What are the five characterization areas?

  • Files and directories

  • The Kernel

  • Processes

  • The network

  • Hardware

e. What is the next step and what is the transition?

Detect; the characterization information has been determined and recorded

1.7. For the Detect Step:

a. What is the purpose?

Find deviations from the expected or characterized step

b. What are the products?

A computer system or network infrastructure component that has deviated from the expected state or for which some external stimulus that affects the system has been recognized.

c. What is being examined in this step?

The characterization data produced in the Prepare step with newly created versions of the same information

d. What is the next step and what is the transition?

Respond; the transitions are:

  • Unusual, unexpected, or suspicious behavior

  • Something new has been learned about the system's characteristics

  • Pertinent information from an external source (user report, interaction with another organization, vendor advisory or bulletin)

  • Change in policies or procedures

  • Change in the risk status of information assets residing on the system

1.8. For the Respond Step:

a. What is the purpose?

To respond to the action that caused the transition from the Detect step

b. What are the products?

A computer system or network infrastructure component where the transition cause has been remediated

c. How do you respond to the following?

  1. An intrusion

    - Further analyze the effects of, the scope of, and the damage caused by the intrusion.

    - Contain these effects as far as possible.

    - Work to eliminate future intruder access

    - Return the system to a known, operational state while continuing to monitor and analyze the system

  2. A patch

    Apply the patch

d. What are some of the issues from the perspective of the SA in the Response process?

Collect and protect information

How long to let the intruder continue to intrude

See the discussion of slide 51 for more issues

e. What is the next step and what is the transition?

Respond; transition is that the system has been returned to a known, operational state

1.9. For the Improve Step:

a. What is the purpose?

The purpose is to improve the process for responding to anomalous behavior.

b. What are the products?

Improved personnel, policies, technology, support for the business case, and correct and appropriate publicity of the event.

c. What is the next step and what is the transition?

Repeat; transition occurs when the personnel, policy, technology, business case, and publicity areas have been addressed.

1.10. What is the purpose?

a. What is the purpose?

The purpose of the Repeat step is to recognize that there have been changes to the computer system or network infrastructure component and that those changes need to be factored into the system's characterization.

b. What are the products?

The product is the recognition that the system needs to be characterized again because something about it has changed.

c. What is the next step and what is the transition?

Prepare; none

1.11. Define the following:

a. Configuration Management

Configuration management is the process by which the configuration of a computer system or network infrastructure component is controlled somehow. Note that the term "Configuration Management" does not mean that the task is done well but only that it is controlled by some technique. It can be done well or badly, in an automated manner or manually.

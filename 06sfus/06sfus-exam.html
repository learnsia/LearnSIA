1. Exam - Introduction

Survivable Functional Units.

Overview: These are the exam questions for Principle 6: Survivable Functional Units (SFUs) are a helpful way to think about an enterprise's networks.

The items that are important in this module are the following:

  • Define Survivable Functional Unit (SFU).

  • Given an SFU, identify the following:

    The information asset(s) it delivers

    The service(s) used to delivery those asset(s)

    The constituency or constituencies it serves

    The authentication scheme it uses

    The attributes that improve its survivability

  • Define how the role of the SFU supports the mission of the enterprise.

  • SAs have to know how to think about improving the survivability of a Functional Unit. That is, they need to at least be able to identify the areas of a Functional Unit that are critical and perhaps single points of failure that need to be addressed through Survivability Engineering. This may be a little more art/common sense than science at this point, although the lecture includes there is the beginning of a process for doing that.

  • How to enumerate options for those areas that have been identified as candidates for survivability improvement.

  • How to assess those options according to some scale, for example financial, implementation effort, on-going maintenance effort. Total cost of ownership is a more inclusive term.

  • How to take the risk methods from Principle 3, Not all data is of equal value to the enterprise ? risk must be managed, and apply them to the enumerated options so that they can select one of them as a candidate for implementation.

  • Know the beginnings of how to create a dependency tree of SFUs in the enterprise. At this point, it is sufficient to indicate what the example SFU depends on, what depends on it, and whether these other SFUs can operate without the example SFU.

  • Be aware of the need to communicate those implementation options and their costs to others in the enterprise.

1.1. Define ?Survivable Functional Unit?

A Survivable Functional Unit is:

  • a collection of one or more computer systems and network infrastructure components (routers, firewalls, hubs, switches, access points, etc)

  • that delivers one or more information assets

  • through one or more services

  • to one or more constituencies

  • authenticated to the Survivable Functional Unit in some fashion

  • and that can survive attacks, outages, errors, etc.

1.2. Explain how the role of the SFU supports the mission of the enterprise.

The key here is being able to define the enterprise network in terms of SFUs and how and what they contribute capabilities to the mission of the enterprise. SFUs provide a clearer understanding of how computer systems and network infrastructure components support that mission.

1.3. Explain the Functional Unit Identification process.

The key here is to be able to use the network captures and other artifacts to decide what computer systems and network infrastructure components there are and then to group them together into Functional Units.

1.4. Describe some ways in which a Functional Unit can be made into a Survivable Functional Unit.

Redundancy, applying patches, using SKiP, using intrusion detection, basically anything that makes the FU survive a level of attacks, outages, and errors better than it could before.

1.5. Name 4 of the attributes that should be evaluated when identifying the options that are available to transform a Functional Unit into a Survivable Functional Unit.

Examples are: capital costs, training costs, maintenance costs, installation costs, support costs.

1.6. Name 4 of the excavation and stones of the roadbed functional unit building blocks.

  1. Domain Name Service

  2. Host Configuration

  3. User Authentication

  4. Logging

  5. Network Intrusion Detection

  6. Network File Service

  7. File Backup and Restore

  8. Administrative

1.7. Give 3 examples of the concrete of the road functional unit building blocks.

  1. Electronic Mail

  2. Web Development and Delivery

  3. Remote Access

  4. Wireless Access

  5. Internet Proxy

1.8. Name 3 of the enterprise artifacts used in the Functional Unit Identification Process.

  1. Network traffic

  2. Logging information

  3. Running processes

  4. Services available to the network

  5. What system administrators think

  6. Infoproc inventory

  7. Policies and procedures that are in place

1.9. Once a set of options for transforming a Functional Unit into a Survivable Functional Unit have been tabulated, what happens next and who is typically involved?

Select from the available options in partnership with SA?s immediate managers. Communicate selection to others in the enterprise based upon the importance of the SFU to the enterprise?s mission using the appropriate communication style.

1.10. What is the goal of the Survivable Functional Unit Dependency Diagram?

To determine which Survivable Functional Units depend on other Survivable Functional Units to better understand how well they can survive.
